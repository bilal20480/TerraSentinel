# TerraSentinel

Hyderabad is sinking. Not dramatically, not all at once — but ward by ward, millimetres at a time, driven by decades of groundwater extraction and rapid construction on compressible soil. The GHMC currently has no live system to track this. By the time subsidence becomes visible — a cracked road, a tilting wall — the damage is already done.

TerraSentinel is a monitoring platform built specifically for GHMC and state disaster management officials. It watches the ground for them, continuously, using radar data from the Sentinel-1 satellite which passes over Hyderabad every 12 days and can detect ground movement down to a few millimetres. That data is processed and run through a fine-tuned TerraMind model that classifies each area by risk level. The output appears on a web dashboard as a live risk map of Hyderabad — wards colour-coded by how fast the ground is moving, with flagged zones that need immediate attention and trend charts showing how displacement has changed over time. The officials who use it don't need to understand satellites or AI. They see the map, they see the flags, they act.

TerraMind is the core of what makes this work. SAR radar data is not intuitive — it doesn't look like a photograph, and interpreting ground movement signals from it requires understanding the relationship between radar backscatter, terrain, land cover, and urban density all at once. We fine-tuned TerraMind-1.0-small on Sentinel-1 acquisitions over Hyderabad because it was pretrained on nine satellite modalities across billions of tokens, meaning it already understood the language of radar imagery before we touched it. That pretrained knowledge is what let us build a useful classifier with a limited local dataset rather than needing years of labelled ground-truth data.

We also used TerraMind's Thinking-in-Modalities feature during fine-tuning, where the model generates a synthetic land-use map internally as a reasoning step before making its prediction. This matters in Hyderabad specifically because the city is dense and mixed — a single ward might have apartment blocks, active construction, and open ground all within a few hundred metres. Without land-use context, a model can confuse construction vibration with real subsidence. TiM gives TerraMind that context without needing a separate land-use input.

The model is evaluated against a baseline of standard InSAR coherence thresholding — the manual method used today. We target a minimum 5% mIoU improvement on a held-out set of Hyderabad patches. TerraMind-small's compact size also means inference is fast and the model is viable for near-edge deployment, where only the risk classification travels down rather than the full raw imagery.

What isn't finished yet is honest to state. Training data currently covers about three months of passes, which isn't enough to account for seasonal ground behaviour. Labels come from processed SAR data rather than physical GPS measurements, which adds some noise. The model has only been trained on Hyderabad and hasn't been tested on other cities. These are the next things to fix — but the core loop works: satellite data in, risk map out, officials informed.
